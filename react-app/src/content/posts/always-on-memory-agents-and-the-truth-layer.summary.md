- The Always-On Memory Agent trades determinism for insight. Its timer-based consolidation actively finds patterns across memories, but runs on LLM inference with no guarantee of consistent results. A truth layer takes the opposite trade: deterministic state, full provenance, no autonomous reasoning.
- The two projects diverge on every design axis: who decides what to store (client agent vs specialist subagents), automatic vs explicit ingestion, LLM-driven vs deterministic extraction, consolidation that mutates memory vs immutable event-sourced truth, single-platform vs cross-platform MCP.
- The clearest distinction is role. The Always-On Memory Agent is an agent with its own reasoning loop. Neotoma is a substrate. When the agent is wrong, having a deterministic layer underneath lets you trace what it read and revert what it wrote.
- The two approaches are complementary, not competing. A consolidation agent reading from and writing to a truth layer via MCP turns consolidation into a non-destructive operation: intelligence above, trust below.
- The open question is not whether agents will remember, but how. The architecture may converge on consolidation agents that think, running on top of truth layers you can trust.
