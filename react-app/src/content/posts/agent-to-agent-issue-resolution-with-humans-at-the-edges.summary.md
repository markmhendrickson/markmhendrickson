- Agent-led evaluation told users whether the product fit. The issues subsystem closes the other half of the loop: when the user's agent hits a bug, notices a performance gap, or identifies a missing capability it wishes existed, it files the report through MCP without leaving the session — without the user ever noticing.
- The MCP instructions tell agents to file immediately when a reportable problem or improvement opportunity is confirmed, without stopping to ask per-issue. The user opts into this at install time — asking "should I file this?" before every report interrupts the session without adding any new information.
- Submission is a single tool call with a schema-enforced reporter environment (git SHA or app version, the latter auto-populated when omitted). The agent can link related entities at submission time so the issue lands already wired into the graph it is about — relationships are created server-side in the same atomic call.
- Threaded follow-up uses a structured message channel between the reporter's agent and the maintainer side. The reporter's agent reads progress using a guest read-back token scoped to the thread it opened.
- On the maintainer side, a triage daemon subscribes to issue creation events through substrate signaling, classifies bugs versus enhancements, and routes accordingly: bugs toward a worktree fix, enhancements toward a plan entity aggregated with related requests and surfaced for human review.
- Enhancements need human judgment in a way bugs don't. A bug has a ground truth. An enhancement is a design claim that carries assumptions about usage patterns and tradeoffs the reporting agent can't fully see. The agent's signal is valuable — it encountered real friction, not imagined friction — but the decision about what to do belongs to a human who can see the full picture.
- This is the cleanest example yet of the nervous system framing in action: the substrate signals on write, consumers decide what to do. The substrate does not prioritize, route, or retry — those are operational-layer concerns.
- Both halves of the loop run as agents talking to agents over one MCP surface. The human never opens a browser tab to evaluate, file, or check status.
- The wire is general. Apps built on Neotoma can let user agents file structured reports against their own entity types and pick them up through subscriptions — inheriting most of the issues subsystem's machinery without rebuilding it.
