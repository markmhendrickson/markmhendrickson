- Three production-scale AI agent platforms (Manus, Claude Code, OpenClaw) independently converged on markdown files as their primary memory system, validating that the agent memory problem is real and felt at billion-dollar scale.
- Both ergonomics (cat, grep, git, editor-open, local-first, debug with `less MEMORY.md`) and economics (Manus's 100:1 input-to-output ratio, the 10x cached-vs-uncached cost spread that makes KV-cache hits a unit economics strategy) are load-bearing — files are a cost-aware ergonomic default, not a simplicity preference.
- The failure modes across all three systems are identical and documented: no versioning, no provenance, no conflict detection, no entity resolution, no schema constraints, and concurrent writes that corrupt state.
- The real competitive incumbent for agent memory is not vector databases — it is markdown files, and the upgrade path runs from files to structured state with integrity guarantees, not from one database product to another; "escalate to Postgres" solves concurrency but leaves versioning, provenance, and entity resolution unsolved.
- The migration can preserve what made files rational: structured state stays canonical while a generated mirror, optional git-per-transaction history, typed editing, bounded `MEMORY.md` export, and one canonical renderer keep directory ergonomics and KV-cache-friendly stable text — details in release notes; the essay claim is that files-as-view beats files-as-source-of-truth for agents that outgrow markdown.
